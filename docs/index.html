01 · O que é

Não é um chatbot. É o seu WhatsApp com um operador.

O WhatsApp Agent não atende os seus clientes com respostas prontas — ele coloca o Claude pra trabalhar pra você, sobre as suas conversas reais, do seu jeito.

📥

Captura tudo

Texto, áudio, imagem, vídeo e documento — cada mensagem que entra ou sai vira registro em Postgres e mídia em Storage, no seu projeto Supabase.

🎙️

Transcreve sozinho

Um cron roda a cada 2 minutos e transcreve os áudios com Whisper. Quando você pergunta "o que a Ana falou?", a resposta já vem em texto.

🧠

Você conversa, ele opera

"Do que eu tô devendo resposta?", "resume a conversa com o Pedro", "responde que fecho amanhã" — linguagem natural, sem comando decorado.

🔒

O banco é seu

Single-tenant por design: cada instalação roda no Supabase de quem instala. Nenhuma mensagem passa por servidor de terceiros do projeto.

02 · Como funciona

Da mensagem ao Claude, em linha reta

Três serviços e nenhum processo local: o provedor recebe a mensagem, o Supabase guarda e executa, e o Claude conversa com tudo isso por MCP.

Provedor WhatsApp

O gateway do seu número: Z-API (hospedado) ou Evolution API (self-hosted), escolhido por instância. Recebe as mensagens e despacha os envios.

Supabase = runtime

Postgres, Storage com 6 buckets de mídia, Edge Functions e pg_cron. Não existe servidor seu pra manter: o projeto Supabase é o backend inteiro.

mcp-api

Uma Edge Function que é um servidor MCP falando HTTP. São 23 ferramentas — inbox, read, send, search e companhia — que qualquer harness MCP consome.

Claude (ou outro)

Claude Code conecta por header key; Desktop e Web por OAuth (a própria mcp-api é o Authorization Server). Funciona em qualquer app com suporte a MCP.

03 · Ferramentas

Você diz. Ele aciona a tool certa.

Nada de comando decorado: são ferramentas MCP que o Claude escolhe sozinho a partir do que você pede, em português.

"Do que eu tô devendo resposta?"inbox
"Resume a conversa com o Pedro" (áudios já transcritos)read
"Responde que fecho amanhã" (confirma antes)send
"Procura onde falaram de orçamento"search
"Categoriza esse chat como cliente"categorize_chat
"Reage com 👍 na última do Pedro"react
"Manda um áudio pra Ana" (texto vira voz)send_voice
"Essa mensagem tá no meu tom?"check_message
E mais: transcrição sob demanda, download de anexos, edição e apagamento de mensagens, anotações por contato, sincronização de grupos, merge de chats duplicados… 23 tools no total — a lista completa está no README.
Pendência sem dívida eterna: o inbox sabe quem deve resposta (waiting_on), grupos ficam fora do cálculo, e quando você sinaliza que uma conversa acabou ("ignora esse", "não precisa responder") o agente marca como resolvida na hora — e ela reabre sozinha se a pessoa voltar a falar. "Me lembra semana que vem" vira snooze.
Um comando, uma chamada: "reage na última do Pedro", "edita minha última mensagem", "apaga", "responde citando" e "encaminha" resolvem o alvo sozinhos (última mensagem, última deles, última sua) — sem precisar ler a conversa antes. A busca por nome cobre o histórico inteiro, e nomes técnicos de contato são curados automaticamente.
04 · Voz

Escrever por você sem parecer um robô

A parte mais difícil de responder em nome de alguém não é o conteúdo — é o tom. O agente resolve isso em duas camadas.

Voice guide global

Um guia do seu jeito de escrever, instalado via setup_voice_guide: saudações que você nunca usa, gírias que usa demais, como você pontua, o que é proibido. Antes de fixar um rascunho, o agente roda check_message — que devolve um score e aponta o que soou artificial.

  • O guia mora no seu banco, versionável no seu repositório.
  • O check pega "fingerprints de IA": hype, afirmação lisa, formalidade que não é sua.

Perfil de voz por contato

Cada relação tem um tom. O voice_profile guarda, por chat, como a pessoa te chama, como você a chama, as gírias dela e o registro da relação — e o agente espelha isso ao redigir, por cima do guide global.

{
  "como_me_chama": ["mano", "irmão"],
  "como_chamo": ["Jão"],
  "girias": ["dale", "top demais"],
  "registro": "informal-íntimo, zoa, áudio longo",
  "confianca": "alta"
}
  • Nasce de um backfill do histórico (skill no repo, com piloto revisável).
  • Se mantém sozinho: captura incremental no uso + rotina semanal local de nutrição.
  • Só evidência literal vira perfil — na dúvida, o contato fica sem perfil e o guide manda.
E a voz literal: com uma conta ElevenLabs, "manda um áudio pra Ana" vira mensagem de voz de verdade — send_voice transforma o texto em PTT usando a voz default da instância (clone da sua voz ou uma voz do acervo, escolhida na instalação).
05 · Provedores

Z-API ou Evolution — você escolhe por instância

Desde a v3, o gateway do WhatsApp é plugável. Os dois convivem no mesmo banco, e instâncias antigas continuam funcionando sem reconfiguração.

padrão recomendado

Z-API

Hospedado (SaaS): você cria a conta, conecta o número por QR code e pronto — sem infra pra manter. Plano pago.

instance_id + auth_token + client_token

self-hosted

Evolution API

Open source e gratuito: você sobe o seu próprio servidor Evolution e aponta o agente pra ele. Pra quem já tem (ou quer) a própria infra.

base_url do servidor + auth_token

06 · Segurança

Single-tenant: o dado não sai do seu controle

Não existe "nuvem do projeto". Cada instalação é o seu número, no seu provedor, gravando no seu Supabase.

🗝️

Segredos só server-side

A secret key do Supabase (que bypassa RLS) e as credenciais do provedor vivem nos secrets das Edge Functions — nunca no repositório, nunca no cliente.

🚪

Acesso com chave

O MCP é protegido por MCP_API_KEY (header x-mcp-key) no Claude Code, ou OAuth com PKCE no Desktop/Web — a própria mcp-api é o Authorization Server.

🧾

Auditável de ponta a ponta

Migrations versionadas, mensagens do agente marcadas (sent_by_agent) e todo envio confirmável antes de sair. O histórico é seu pra auditar.

07 · Instalação

O Claude instala pra você

Dois planos bem separados: um setup que você roda uma vez, e a operação do dia a dia — que não precisa nem do repositório.

Crie as três contas

Supabase (banco + functions), um provedor de WhatsApp (Z-API ou Evolution) e OpenAI (Whisper pra transcrição). Os planos gratuitos dão pra começar.

Clone o repo e rode a skill /setup

Abra a pasta no Claude Code (aba Code do Claude Desktop) e peça o setup: o Claude instala o Supabase CLI se faltar, pede uma credencial de cada vez (só no .env local, gitignored), aplica as migrations, configura os secrets e deploya as Edge Functions.

Conecte o MCP e opere

Aponte o seu harness pra https://SEU_PROJECT_REF.supabase.co/functions/v1/mcp-api — uma vez. Daí em diante é conversa: inbox, resumos, respostas no seu tom. Recomendado: rode o backfill do perfil de voz e deixe a rotina semanal de nutrição agendada.
